WEB TESTING WORKFLOW

1. Source review first:
   - Trace request routes to sensitive sinks.
   - Identify auth/session model.
   - Identify storage model and seed data.
   - Locate input validation and trust boundaries.
   - Locate flag access path.

2. Dynamic recon:
   - Use curl/httpie/Python requests.
   - Save interesting responses under artifacts/.
   - Enumerate routes from source before fuzzing blindly.
   - Inspect cookies, JWTs, CSRF tokens, redirects, CORS, headers, static JS, sourcemaps.

3. Exploit development:
   - Build minimal proof-of-concept requests.
   - Convert successful manual steps into scripts/solve.py.
   - Make HOST, PORT, BASE_URL, SCHEME configurable at the top or via env vars.
   - Add clear comments only for non-obvious exploit steps.

4. Remote validation:
   - Run scripts/solve.py against the local service first if available.
   - Then run it against BASE_URL.
   - Handle differences: host validation, JWT audience, SAML fingerprint, cookie domain, path prefix, reverse proxy headers, timing, rate limit, different seed data.
   - Keep iterating until the remote response proves the real flag.

WEB SKILL CHECKLIST

Apply relevant techniques:
- Auth bypass: weak sessions, predictable tokens, JWT alg/secret/kid/jku, OAuth redirect_uri, SAML XML signature wrapping, default admin, IDOR.
- Injection: SQLi, NoSQLi, command injection, SSTI, LDAP/XPath, GraphQL injection, XXE.
- File bugs: path traversal, arbitrary file read/write, upload extension bypass, symlink/zip-slip, image parser abuse.
- SSRF: cloud metadata, internal services, localhost admin panels, DNS rebinding, redirects, URL parser confusion.
- Deserialization: pickle, YAML unsafe load, PHP object injection, Java serialization, Node serialize.
- Client-side: JS source maps, prototype pollution, DOM XSS, localStorage secrets, WebAssembly hints.
- Infrastructure: nginx alias traversal, proxy trust headers, cache poisoning, request smuggling, CORS.
- Race/logic: coupon/order logic, reset token races, TOCTOU, inconsistent validation between frontend/backend.
